# 谁是小丑 - Who Is The Clown

仿CF穿越火线"谁是小丑"模式的第一人称3D网页游戏，手机触控操作。

## 技术栈
- **React 18** + **Three.js r128**
- 纯前端，Vite构建
- 当前为单机版，后续计划WebSocket多人联机

## 快速开始
```bash
npm install
npm run dev
```

## 项目结构
```
src/

```

## 已实现功能

### 核心玩法
- 第一人称视角，手机触控（左摇杆移动，右侧滑动转视角）+ 桌面WASD+鼠标
- 两个阵营：小丑 vs 警察
- 玩家初始为小丑，被击杀后变身警察继续游戏
- 小丑一击必杀警察（近战），警察需要两枪杀死真小丑
- 6个钱包点分散在地图四角，占领5个获胜，占领需3秒读条

### AI系统
- **AI小丑**（10个，其中3个是"真小丑"）：
  - 4种伪装行为：idle/wander/zigzag/lookaround
  - 真小丑额外行为：steal(偷钱包)、capturing(读条中)、hunt_cop(追杀警察)
- **警察AI**（3个）：
  - patrol(巡逻)、suspicious(观察)、chase(追玩家)、hunt_clown(主动搜杀小丑)
  - 70%概率正确识别偷钱包的真小丑，15%误杀普通AI
  - 误杀扣30HP，误杀过多会自己死亡
- 卡墙检测：卡住0.5秒自动换路线

### UI/HUD
- HP、时间、钱包数、存活数、阵营标签
- 击杀播报（左侧feed）、钱包进度条（右上5格）
- 小地图（建筑、钱包状态、玩家位置。警察模式额外显示队友）
- 准星、占领进度条

### 特殊机制
- 疯狂时刻（最后60秒，大量AI涌入）
- 白天场景、人形角色模型、走路动画

### 胜负条件
- 小丑胜：占5钱包 / 消灭所有警察
- 警察胜：消灭所有真小丑 / 时间到钱包没占满
- 变身：小丑被杀→警察(50HP)

## 已知问题

### Bug
1. 碰撞只有AABB，偶尔穿模
2. 射击判定用dot product不够精确（角度阈值0.82）
3. setHud每帧调用，低端机可能卡

### 待优化
1. 无音效（枪声、脚步、占领、击杀）
2. 无射击动画（枪口闪光、弹道）
3. 无受伤反馈（屏幕闪红、击退）
4. 地图太空旷，需要更多掩体
5. 建筑只是方块
6. 无开始倒计时
7. 角色模型可以更精细

### 功能扩展方向
1. 卡牌系统（隐身/烟雾/回血/雷达）
2. 多张地图
3. 难度选择
4. 数据统计

## 多人联机计划

架构：`React+Three.js <--WebSocket--> Node.js`

免费服务器：Cloudflare Workers / Render.com / Fly.io

需要改的核心点：房间系统、角色分配、状态同步、事件广播、延迟补偿、断线重连

## 配置参数（CFG对象在Game.jsx顶部）

| 参数 | 值 | 说明 |
|------|-----|------|
| MAP | 60 | 地图大小 |
| WALK/SPRINT | 4/7 | 走路/疾跑速度 |
| AI | 10 | AI小丑数（前3个是真小丑）|
| COPS | 3 | 警察数 |
| WIN_W | 5 | 需占领钱包数 |
| COL_T | 3 | 占领耗时(秒) |
| TIME | 240 | 总时间(秒) |
| RAGE_T | 60 | 疯狂时刻触发(剩余秒) |
| PV/PA | 12/π*.55 | 警察视野距离/角度 |
| PSD | 20 | 警察每枪伤害 |
| PMK | 30 | 误杀AI惩罚HP |
| AD | 100 | 小丑攻击伤害(一击必杀) |
